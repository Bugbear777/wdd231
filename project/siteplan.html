Board Game Hub

A compact recommendation site for casual and hobbyist board gamers — site plan.

Site Name

BoardGameHub

Why this name: "BoardGameHub" is short, searchable, and clearly communicates the site’s core purpose: a single place (a hub) where players can discover, filter, and pick board games for different groups and occasions.

Optional domain (example): boardgamehub.dev — domain availability is not required for class but this shows a plausible brand domain.

Site Purpose

BoardGameHub provides a lightweight recommendation tool for people deciding what board game to play. The site will include:

  • Homepage: friendly intro to board gaming and quick links to the main tools.
  • Game Finder: searchable & filterable list of games stored in a local JSON file (player count, playtime, difficulty, genre tags).
  • Random Game Picker: a JavaScript tool that picks a random game from the same list to solve the "what should we play?" problem fast.

Additional capabilities (stretch goals): favorites stored in localStorage, sort-by options, small integration with an external API like Board Game Atlas for live data.

User Scenarios

These are example questions (scenarios) a target visitor would ask the site. They drive the content and UI priorities.

  1. “We have 4 players and only 45 minutes — what games fit that?”

    This scenario demands accurate filtering by player count and playtime and a clear sort option.

  2. “I want a light, family-friendly cooperative game for 2–5 players.”

    This drives the need for tags (cooperative, family) and adjustable difficulty labels.

  3. “I don’t care — surprise us!”

    Maps to the Random Game Picker tool that chooses from the filtered list or entire collection.

Where each color is used

  • --primary (#1F7A5A): page headings, primary CTA buttons, active nav states, logo background.
  • --accent (#F0C808): badges (e.g., "Quick Pick"), highlights, promotional banners, small icons.
  • --muted (#5B6B63): secondary text, meta info (playtime, player count), form helper text.
  • --bg & --card-bg: page background and white content cards for contrast and readability.

Typography

No more than three fonts — chosen for readability and a friendly, modern aesthetic.

Playfair Display — Headings

Use for: main headings (H1, H2) and hero titles to add a touch of character.

Inter — Body & UI

Use for: body paragraphs, navigation, form inputs, and labels. Inter is readable at small sizes and works well for UIs.

Font choices applied in this plan: Playfair Display for headings and Inter for body text (both loaded from Google Fonts at the top of this document).

CSS & Implementation Notes

  • All site pages will reuse the same color variables defined at the top to ensure consistent theming.
  • Typography: Google Fonts loaded in the document head; use CSS variables to switch weights for headings and body text.
  • Styling approach: a small global CSS file (or a few components) is sufficient — keep styles modular (header.css, finder.css, picker.css) if splitting files for the project.
  • Data: initial development will use a local games.json file with ~20 sample games (fields: id, name, minPlayers, maxPlayers, playtimeMin, playtimeMax, difficulty, tags, thumbnail).
  • JavaScript: DOM-driven filtering and the Random Picker. Consider small utility functions for filtering, sorting, and localStorage favorites.
  • Accessibility checklist: keyboard navigation for filters, semantic HTML (<form>, <label>), ARIA attributes for dynamic components, and color contrast tested to WCAG AA where possible.
